Overview

Built a full-stack web application that takes budget and dietary constraints as inputs and generates matching recipes via a custom filtering algorithm. The app was containerized with Docker and deployed on Microsoft Azure, with user testing informing two major UX iterations that improved navigation efficiency by 20%.

Core Concepts

  • Constraint-based filtering algorithm — matches ingredients to recipes based on budget ceiling, dietary restrictions, and pantry availability
  • User testing and gap analysis — conducted testing sessions to identify UI friction points, leading to documented UX improvements
  • Containerized deployment — Docker image deployed to Azure App Service for scalable, reproducible hosting
  • Full-stack architecture — Python backend with a JavaScript/HTML/CSS frontend, REST API connecting the two layers

Development Notes

The core filtering algorithm scores recipes against user constraints using a weighted match function, returning ranked results:

def filter_recipes(budget, pantry, restrictions, recipes):
    results = []
    for recipe in recipes:
        score = compute_match(
            budget=budget,
            ingredients=recipe["ingredients"],
            pantry=pantry,
            restrictions=restrictions
        )
        if score > THRESHOLD:
            results.append((score, recipe))
    return sorted(results, reverse=True)
