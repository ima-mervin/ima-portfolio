Overview

Designed and automated Oracle FDI dashboards for Liberty Energy's HR reporting infrastructure, covering 5,800+ employees. The project centered on resolving longstanding historical headcount metric errors caused by inconsistent data entry and schema drift across reporting periods, then standardizing the output for executive consumption.

Core Concepts

  • Requirements gathering from HR and finance stakeholders to define correct headcount logic and metric definitions
  • Custom calculated fields in Oracle FDI to reconcile historical discrepancies across fiscal periods
  • Automated refresh pipeline so dashboards update without manual intervention each reporting cycle
  • Executive-ready layout — KPIs surfaced at the top, drill-down by department, location, and tenure

Development Notes

The root cause of metric errors was a mismatch between headcount snapshot dates and payroll cycle cutoffs. The fix involved a corrective SQL layer that normalized the date spine before aggregation:

-- Normalize headcount to period end date
SELECT
  employee_id,
  department,
  LAST_DAY(hire_date) AS period_end,
  COUNT(*) OVER (PARTITION BY department) AS headcount
FROM hr_employees
WHERE status = 'ACTIVE'
ORDER BY period_end DESC;
"Clean data at the source eliminates a thousand downstream corrections."

Once the corrective layer was in place, the dashboards were connected directly to the normalized view, ensuring all future reporting automatically reflected accurate headcount figures without manual reconciliation.
